NLP

NLP allows computers to analyze and derive meaning from human language. Using NLP knowledge from text can be organized and structured to perform tasks including automatic summarization, translation, named entity recognition, relationship extraction, sentiment analysis, speech recognition, and topic segmentation.

But ML algorithms can't process text data; they need some kind of numerical feature vector. So the first step in any kind of NLP exercise, requires preprocessing text to make it suitable for ML. Basic text pre-processing includes:

  • case normalization: converting all text into uppercase/lowercase, so the algorithm is case agnostic
  • tokenization: converting normalized text into a list of tokens. Sentence tokenization is used to generate a list of sentences, while word tokenization is used to generate a list of words.

NLTK includes a pre-trained Punkt tokenizer for English.
Text preprocessing steps involves the following:

  • removing noise: anything that isnâ€™t a standard number or letter
  • removing stop words: very common words that add little value in analysis are removed from the vocabulary.
  • stemming: reducing inflected (or derived) words to their stem, base or root formâ€Š
  • lemmatization: similar to stemming, however stemming can often create non-words, whereas lemmas are actual words

Bag of Words (BoW) Model
After preprocessing, the text is transformed into a meaningful vector of numbers. BoW is a representation of text describing the occurrence of words within a document, and involves 2 things:

  • a vocabulary of known words
  • a measure of the presence of known words

It's called a â€œbag of words" because any information about the order or structure of words in the document is discarded. The model only cares whether the known words occur in the document, but not where they occur.
Intuitively, documents are similar if they have similar content.

For example: Suppose we have dictionary containing words {Learning, is, the, not, great}, and we want to vectorize the text â€œLearning is greatâ€. Its vector representation would be : (1, 1, 0, 0, 1), where the numbers represent their word counts.

Term Frequency-Inverse Document Frequency (TF-IDF)
A problem with BoW is that highly frequent words start to dominate the document (e.g. larger score), but such words may not contain much informational content. It also gives more weight to longer documents than shorter documents.

One approach is to rescale the frequency of words by how often they appear in all documents. The scores for frequent words that are also frequent across all documents are penalized. This scoring is called Term Frequency-Inverse Document Frequency, where

Term Frequency: a scoring of the frequency of the word in the current document

  • TF = (Number of times term t appears in a document)/(Number of terms in the document)
Inverse Document Frequency: a scoring of how rare a word is across documents.
  • IDF = 1+log(N/n), where, N is the number of documents and n is the number of documents a term t has appeared in.


Cosine Similarity: is a measure of similarity between two non-zero vectors of an inner product space.

  • Tf-idf weight is a weight often used in information retrieval (IR) and text mining. It is a statistical measure used to evaluate how important a word is to a document in a collection or corpus
  • Cosine Similarity (d1, d2) = Dot product(d1, d2) / ||d1|| * ||d2|| where d1,d2 are two non zero vectors.

stop words and puntuationÂ¶

Punctuation and frequent words like "the" or "that" are not useful for classification. These are referred to as stopwords. NLTK has a a stop words corpus which can be downloaded: nltk.download("stopwords")

In [59]:
from nltk.tokenize import word_tokenize
from nltk.stem import PorterStemmer
import string

# list of stop words and punctuation
useless_words = nltk.corpus.stopwords.words("english") + list(string.punctuation)

def preprocess_words(words):

    # filter out stop words/punctuation from all movie reviews
    filtered_words = [word for word in words if not word in useless_words]

    # stem words
    stemmer = PorterStemmer()
    filtered_words = [stemmer.stem(word) for word in filtered_words] 
    return filtered_words
    

filtered_words =  preprocess_words(movie_reviews.words())

Naive Bayes Classifier is a simple classifier with a probabilistic approach to classification. The relationships between the input features and the class labels is expressed as probabilities. Given input features, the probability for each class is estimated, where the class with the highest probability determines the label for the sample.

bag-of-words modelÂ¶

The simplest model for analyzing text is bag-of-words.

With BoW text within a document, is converted into a matrix of word occurrence. The model is concerned about whether or not a word occurres in the document.

import numpy as np
import pandas as pd

from nltk.corpus import stopwords
from nltk.stem.wordnet import WordNetLemmatizer
import string
import re
#from collections import Counter
 
stop = set(stopwords.words('english'))
exclude = set(string.punctuation)
lemma = WordNetLemmatizer()
 
# Cleaning the text 
def clean(doc):
    stop_free = " ".join([i for i in doc.lower().split() if i not in stop])  # lower-case. stop words removal
    punc_free = ''.join(ch for ch in stop_free if ch not in exclude)         # punctuation
    normalized = " ".join(lemma.lemmatize(word) for word in punc_free.split()) # lemmatization
    processed = re.sub(r"\d+","",normalized)
    y = processed.split()
    return y

def getCleanText(doc):
    clean_lines = []
    for line in doc:
        line = line.strip()
        cleaned = clean(line)
        cleaned = ' '.join(cleaned)
        clean_lines.append(cleaned)
    
    return clean_lines

Tokenizing textÂ¶

Tokenizing and filtering of stopwords are included in CountVectorizer, which builds a dictionary of features and transforms documents to feature vectors. However longer documents will have higher average count values than shorter documents. The tf-idf weight is a statistical measure to evaluate the importantance of a word in a document from a corpus. Importance increases proportionally to the number of times it appears in the document but is offset by the frequency of the word in the corpus. Term frequency (TF) is how common a word is, inverse document frequency (IDF) is how unique or rare a word is.

A word about working with imbalanced data sets

Refer to supplemental material for this week.

Most ML algorithms do best with balanced data as most are designed to maximize accuracy and reduce error. With a highly imbalanced dataset, Accuracy is not a reliable performance metric. Take an extreme case of 1:100 where the minority class is 1 while the majority class is 100. Accuracy (correct predictions/all predictions) will be high, but that is clearly misleading. For cases of imbalance (typical of spam, fraud, medical diagnosis), we prefer good performance on the minority class over good performance on both classes.

When deailing with severe imbalance, other metrics are better including:

  • Precision - It is a measure of a classifierâ€™s exactness. Low precision indicates a high number of false positives.
  • Recall- measure of a classifierâ€™s completeness. Low recall indicates a high number of false negatives.
  • F1 Score: balances both precision and recall (the weighted average of precision and recall

Weighted learning algorithms that increase the cost of classification mistakes on the minority class.In scikit-learn, a number of ML algorithms provide an argument class_weight='balanced' which if set, will penalize mistakes on the minority class by an amount proportional to how under-represented it is.

Other approaches include using other algorithms that 'may' work better for imbalanced dataset

  • Decision trees such as RandomForest work by learning a hierarchy of if-else questions and can force both classes to be addressed.

Balance the dataset using Resampling or generating synthetic samples

  • Oversampling: adding more copies of the minority class. Can be a good choice when you donâ€™t have a lot of data
  • Undersampling: removing some observations of the majority class. Can be a good choice when you have a a lot of data (millions)
  • SMOTE (Synthetic Minority Oversampling Technique) uses a nearest neighbors algorithm to generate new and synthetic data for use in training
